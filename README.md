# UmBala — детский тренажёр по школьным предметам (MVP)

Офлайн-приложение на Python / Kivy / KivyMD. MVP покрывает математику и
чтение для 1–4 класса, интерфейс на русском, казахском и английском.

## Структура проекта

```
edukids/
├── main.py              # экраны, навигация, UI (KivyMD)
├── database.py          # SQLite: профили, монеты/XP, прогресс по темам
├── content_loader.py    # загрузка тем/заданий из content/*.json
├── gamification.py      # правила начисления монет, XP, звёзд, значков
├── localization.py      # словарь переводов ru / kk / en
├── creatures.py         # векторные "живые" аватары (см. ниже)
├── content/
│   ├── math.json         # темы и задания по математике, 1-4 класс
│   ├── reading.json       # темы и задания по чтению, 1-4 класс
│   └── shop.json          # каталог магазина: аватары/аксессуары/комнаты
├── requirements.txt
└── buildozer.spec        # конфиг для сборки Android APK
```

## Запуск на Windows/десктопе

```
pip install kivy kivymd
python main.py
```

При первом запуске приложение попросит имя ребёнка и класс — дальше
профиль сохраняется в `edukids.db` (SQLite, создаётся автоматически
рядом с проектом).

## Предметы

Пять предметов: **Математика, Чтение, Қазақ тілі, Английский, Познание мира**.
Все грузятся автоматически через `content_loader.py` - для добавления
нового достаточно положить `content/<subject>.json` того же формата
(см. "Как добавить новый предмет" ниже), правки кода не нужны, только
одна строка в списке плиток `HomeScreen.refresh()` + перевод названия.

- **Қазақ тілі / Английский** — лексика (переводы слов ru↔kk и ru↔en) по
  категориям: цвета, числа, семья, животные (1 кл.), еда, части тела,
  школьные вещи, дни недели (2 кл.), действия, погода, время, профессии
  (3 кл.), эмоции, природа, повтор профессий/животных (4 кл.). Специально
  ограничено словарным запасом (а не грамматикой/падежами) - лексику я
  могу гарантировать точно, а вот сложные грамматические правила легко
  дать с ошибкой, что хуже, чем не дать вовсе.
- **Познание мира** — природа, география, факты о Казахстане: времена
  года и дикие/домашние животные (1 кл.), растения/правила безопасности/
  мой Казахстан/части суток (2 кл.), солнечная система/круговорот воды/
  животный мир Казахстана/полезные ископаемые (3 кл.), материки и океаны/
  экосистемы/экология/география Казахстана (4 кл.). Факты о Казахстане
  намеренно ограничены общеизвестными и безопасными (столица, валюта,
  флаг, крупные реки/озёра) - без политически чувствительных тем.

Итого **2456 заданий в 84 темах** по всем пяти предметам вместе.

## Система уровней (последовательная разблокировка)

Темы внутри каждого класса+предмета теперь называются "Уровень 1, 2, 3…" и
открываются строго по порядку — `TopicListScreen.open_for()` в `main.py`:

- Уровень 1 в классе всегда открыт.
- Уровень N (N>1) открывается, только если пройден (получена хотя бы 1
  звезда) уровень N-1.
- Пройденные уровни остаются доступны для повторного прохождения (со
  звёздами) - каждый повтор берёт новую случайную выборку из банка
  заданий темы (см. `LessonScreen.SESSION_SIZE`), поэтому не повторяется
  один в один.
- Заблокированные уровни показаны серым с иконкой замка, без возможности
  открыть.

Так как это встроено в существующую структуру тем (уже отсортированы по
возрастанию сложности - счёт → сложение → умножение → дроби и т.д.), уровни
не требуют отдельного файла контента - это просто новый способ показа и
разблокировки уже существующих тем. Если нужна отдельная сущность "уровень"
поверх нескольких предметов сразу (не по одному предмету за раз) - это
следующий шаг, отдельная надстройка над текущей структурой.

## Недавно исправленные баги (продолжение)

- **Число, вводимое ребёнком, всё ещё было не видно** - предыдущий фикс
  (`text_color_normal`) на самом деле ни на что не влиял: в KivyMD видимый
  текст в поле ввода управляется свойством `foreground_color`
  (`self.theme_cls.text_color` по умолчанию в файле `textfield.kv` самой
  библиотеки) - `text_color_normal`/`text_color_focus` управляют чем-то
  другим. В моём тестовом окружении цвет темы случайно был тёмным (поэтому
  тесты показывали "исправлено"), а в реальном окружении - нет. Теперь
  `foreground_color` задаётся явно и жёстко, независимо от темы.

## 7 класс - алгебра и геометрия

Математика 7 класса поделена на два блока тем внутри того же предмета
`math` (не стал заводить отдельные предметы "Алгебра"/"Геометрия" -
избыточно для текущей структуры, но темы явно называются по разделам):

- **Алгебра**: одночлены и многочлены (сложение подобных слагаемых,
  умножение), линейные уравнения (включая переменную в обеих частях),
  степень с натуральным показателем (правила умножения/возведения
  степеней), линейная функция y = kx + b.
- **Геометрия**: углы и их измерение (острый/тупой/смежные/сумма на
  прямой), треугольники (классификация, сумма углов = 180°), параллельные
  прямые и секущая (накрест лежащие/соответственные/вертикальные углы).

Алгебраические генераторы (`gen_linear_equations`, `gen_powers` и т.д.)
проверены дважды: автоматическим валидатором структуры и независимым
пересчётом случайной выборки по регулярным выражениям (см. историю
разработки) - ответы совпали на 100% проверенных примеров.

Плюс продолжены казахский/английский (здоровье, экология, общение, черты
характера) и чтение (факт и мнение, аргументация, цель автора текста) для
7 класса.

## Естествознание (5-6 класс)

Продолжение «Познание мира» для 5-6 класса под предметом `world` -
название автоматически меняется на «Естествознание» (топбар и плитка на
главном экране), начиная с 5 класса, как в реальной программе -
`TopicListScreen.open_for()` и `HomeScreen.refresh()` проверяют
`grade >= 5`.

Темы:
- **5 класс**: тела/вещества/явления, живая и неживая природа (тема
  живая/неживая расширена шаблоном-классификацией до 35 заданий), клетка -
  единица живого, простые механизмы.
- **6 класс**: смеси и чистые вещества, системы организма человека,
  пищевые цепи и приспособления, свойства воды и воздуха.

Как и с фактами о Казахстане ранее - специально ограничился
общепринятыми, проверяемыми школьными фактами (агрегатные состояния
вещества, простые механизмы, системы организма на базовом уровне) без
сложной терминологии, которая появляется в специализированных предметах
(физика/химия/биология) с 7 класса.

## Классы 5-11 (в процессе)

Выбор класса (в приветственном экране и в профиле) теперь работает на
диапазон 1-11 (прокручиваемая панель кнопок). Контент пока наполнен для
**5 и 6 класса** (математика, казахский, английский, чтение) - это
естественное продолжение начальной школы, безопасное для генерации без
экспертной проверки. Для классов без контента (сейчас 7-11) экран темы
показывает дружелюбное сообщение "темы скоро появятся" вместо пустого
экрана или краша - `TopicListScreen.open_for()` проверяет пустой список.

**Почему не сразу 7-11**: с 7 класса в реальной программе математика
делится на алгебру и геометрию, литература требует разбора конкретных
произведений, появляются отдельные предметы (физика, химия, биология,
история) вместо "Познание мира" - это требует более вдумчивой проработки
программы, чем механическое продолжение шаблонов 1-6 класса. Специально
не стал спешить с этим, чтобы не выдать поверхностный/ошибочный контент
по сложным темам.

Наполнено для 5-6 класса:
- **Математика**: 10 новых тем по 50 заданий (десятичные дроби, проценты,
  отрицательные числа, скобки, площадь/объём - 5 класс; пропорции,
  смешанные дроби, уравнения, координатная плоскость, проценты в задачах -
  6 класс) = 500 заданий.
- **Қазақ тілі / Английский**: по 8 новых тем (дом, одежда, спорт,
  транспорт - 5 класс; хобби, технологии, география, продвинутые глаголы -
  6 класс) = ~570 заданий на каждый язык.
- **Чтение**: 5 новых тем (эпитет/сравнение, тема и план текста, прямая/
  косвенная речь - 5 класс; метафора/олицетворение, жанры литературы -
  6 класс) = ~57 заданий.

## Контент по темам (1-4 класс)

**1452 задания** в 36 темах (было 108). Большая часть математики и часть
чтения сгенерирована процедурно и автоматически проверена (уникальность,
существование иконок, правильный ответ входит в варианты, перевод на все
3 языка) - это не просто вручную напечатанный текст, а математически
пересчитанные примеры.

- **Математика**: ровно 50 заданий в каждой из 20 тем (1000 всего) -
  генерируются скриптом (см. `/tmp`-скрипты генерации, не входят в финальный
  пакет) на основе арифметических правил, поэтому вопрос вида "23 + 45 = ?"
  не нужно переводить - это просто числа. Для тем со словами (сравнение,
  время, текстовые задачи) используются переведённые шаблоны фраз с
  подстановкой случайных чисел/имён.
- **Чтение**: механические навыки (буквы, слоги, части речи, синонимы/
  антонимы, короткие предложения) тоже доведены до 45-50 через банки слов.
  Темы, которым нужен связный текст (главная мысль, причина-следствие,
  сравнение текстов, пересказ, пунктуация) расширены вручную в 5-7 раз
  (с 3 до 7-18 заданий на тему) - полные полсотни разных, но осмысленных
  коротких историй на такую тему сложнее написать без потери качества за
  один присест; это следующий кандидат на доработку, если нужно больше.

**Важно про размер банка vs размер урока**: 50 заданий в теме - это
*банк*, из которого при каждом открытии темы случайно выбирается **8**
заданий на сессию (`LessonScreen.SESSION_SIZE`) - иначе ребёнку пришлось
бы решать все 50 подряд за один раз. Благодаря банку тема не повторяется
одинаково при следующем заходе.

## Как устроена игровая механика

- За каждое верное задание — монеты (`gamification.COINS_PER_CORRECT_FIRST_TRY`)
  и опыт (`XP_PER_CORRECT`). Меньше монет, если ответили не с первой попытки —
  это мотивирует думать, но не наказывает за ошибку.
- Уровень растёт по формуле `100 * level` опыта на переход — числа маленькие,
  ребёнку легко считать прогресс самому.
- За полное прохождение темы — бонус и 1–3 звезды в зависимости от точности.
- Значки (`gamification.BADGES`) выдаются через `database.grant_achievement`,
  логика "когда выдавать" пока minimal (за первое полное прохождение темы) —
  это первое место для расширения.

## Как добавить новый предмет (например, «Қазақ тілі» или «Познание мира»)

1. Создать `content/kz_tili.json` по образцу `content/math.json` —
   структура `grades -> "1".."4" -> topics -> tasks`.
2. Добавить перевод названия предмета в `localization.py`
   (`subject_kz_tili`).
3. Добавить плитку предмета в `HomeScreen.refresh()` в `main.py`
   (один элемент в списке `subjects`).
Логика загрузки, экраны заданий и геймификация уже общие — их трогать не нужно.

## Как добавить новый класс (5-11)

Просто добавить ключ `"5"`, `"6"` и т.д. в `grades` внутри JSON-файла
предмета. `WelcomeScreen` пока ограничивает выбор классом 1-4 — при
расширении на 5-11 нужно поменять диапазон в `WelcomeScreen.on_kv_post`.

## Магазин, инвентарь и экипировка

- **Магазин** (`ShopScreen`) — покупка новых вещей за монеты. Три категории:
  аватары, аксессуары, комнаты (фон главного экрана). Каталог —
  `content/shop.json`, добавлять новую вещь = дописать один объект.
- **Инвентарь** (`InventoryScreen`) — отдельный экран, показывает только уже
  купленные вещи и позволяет переключаться между ними (без цен и кнопки
  "Купить"). Открывается иконкой сундука в нижней панели главного экрана.
- Аватары и комнаты всегда должны быть каким-то одним (нельзя "снять"
  совсем) — переключаются между собой. Аксессуары можно снять полностью
  (кнопка "Снять").
- Общая карточка товара вынесена в `build_item_card()` в `main.py` — она же
  используется и в магазине, и в инвентаре, чтобы не дублировать разметку.
- Покупка/владение/экипировка — в `database.py` (`purchase_item`,
  `equip_avatar`, `equip_room`, `get_owned_item_ids`). Один и тот же товар
  нельзя купить дважды, при нехватке монет возвращается статус
  `not_enough_coins`.

## Недавно исправленные баги (техническая память проекта)

- **Аватар "убегал" в угол окна.** Причина: `AvatarButton` был на
  `FloatLayout`, а не `RelativeLayout`. У `FloatLayout` дети без `pos_hint`
  позиционируются в АБСОЛЮТНЫХ координатах окна, а не относительно самого
  контейнера - поэтому дочерний виджет с `pos=(0,0)` рисовался в углу
  всего окна, а не в углу своего маленького контейнера в шапке. Исправлено
  переходом на `RelativeLayout` (см. класс `AvatarButton` в `main.py`).
  Тот же класс проблем - смотреть первым делом при любых будущих багах
  "виджет не там, где должен быть".
- **Текст в поле ввода не было видно.** `MDTextField.text_color_normal` по
  умолчанию `[0,0,0,0]` (полностью прозрачный!) в этой версии KivyMD. Нужно
  явно задавать `text_color_normal`/`text_color_focus` - иначе поле работает
  корректно, но введённый текст невидим.
- **Урок требовал решить весь банк (50) за один присест.** Исправлено -
  `LessonScreen` теперь берёт случайную выборку (`SESSION_SIZE = 8`) из
  банка при каждом открытии темы.

## Дизайн главного экрана

Приведён ближе к первоначальному макету: белый кружок-подложка под
аватаром в шапке, карточка "квест дня" персикового цвета с иконкой в
цветном кружке (а не просто строка текста на фоне шапки), у каждого
предмета — цветной кружок-бейдж с иконкой и тонкий прогресс-бар
(пройдено тем / всего тем) под названием.

## Живые персонажи (аватар)

Вместо плоских иконок и вместо эмодзи аватар нарисован как векторная
разноцветная "зверушка" — `creatures.py`, класс `CreatureWidget`. Рисуется
примитивами `kivy.graphics` (Ellipse/Triangle/Line), а не шрифтом иконок и
не PNG-картинками — поэтому гарантированно одинаково выглядит и на
Windows, и на Android, без какой-либо возни с ассетами/кэшем (та же
проблема, что была в FlowDo с PNG-кэшем эмодзи, здесь решена в корне).

13 видов: `cat, dog, owl, koala, panda, rabbit, robot-happy,
unicorn-variant, alien, sword-cross (дракон), duck, fish, bee` — те же
ключи, что и имена иконок в `content/shop.json`, так что каталог менять не
пришлось. Каждый вид — запись в таблице `SPECIES` в `creatures.py`
(основной/дополнительный цвет + форма ушей/носа/клюва), плюс несколько
видов с уникальной отрисовкой (панда, пчела, рыбка, дракон, единорог,
робот) вне общего шаблона.

Аватар находится в шапке главного экрана (и крупнее — в профиле),
покачивается (`_start_avatar_idle_animation`) и время от времени сам
делает "трюк" — кувырок с подпрыгиванием (`play_avatar_trick`, каждые 18
секунд через `Clock.schedule_interval`). Обёрнут в `Scatter` — через него
удобно программно крутить/масштабировать для трюка, не мешая тапу
(тап по аватару открывает магазин).

Полный список доступных имён иконок Material Design (для quest_icon и
магазина, не для аватаров): `from kivymd.icon_definitions import md_icons`.

## Комната (фон главного экрана)

- Список комнат с цветом — `content/shop.json` -> `"rooms"`. У комнаты нет
  иконки, только `color` (RGBA 0-1), в магазине показывается цветным
  прямоугольником-превью.
- Фон применяется в `HomeScreen._apply_room_background()`: canvas-инструкции
  (`Color` + `Rectangle`) пересоздаются заново при каждом обновлении экрана,
  а не мутируют уже отрисованный виджет — см. раздел про рендеринг ниже.

## Смена класса

- В профиле (`ProfileScreen`) есть ряд кнопок 1-4 — текущий класс подсвечен.
  Смена класса пишется в БД (`database.update_grade`) и сразу отражается на
  главном экране (темы, квест дня) при следующем заходе на "Дом".
- Если класс пройден — можно перейти на следующий; если нужно повторить
  прошлый материал — можно вернуться на класс назад. Прогресс по темам не
  теряется при переключении (хранится отдельно по `subject`+`topic_id`).

## Важный технический момент про рендеринг (KivyMD 1.2.0)

В этой версии KivyMD виджеты не всегда перерисовываются, если просто
поменять их свойство (`.icon`, `.md_bg_color`) уже после первой отрисовки —
особенно после возврата с другого экрана. Рабочий паттерн, использованный
везде в проекте: **не мутировать свойство существующего виджета, а удалять
его и создавать заново** (`clear_widgets()` + новый виджет) при каждом
обновлении экрана. Так сделаны аватар, карточки предметов/тем/магазина и
фон комнаты (через пересоздание canvas-инструкций). Если будете добавлять
новые динамические элементы — используйте тот же подход.
